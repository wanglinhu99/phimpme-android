    Phimp.me AndroidÂ¶

    Phimp.me is an Android image editor app that aims to replace proprietary photographing and image apps on smart phones. It offers features such as taking photos, adding filters, editing images and uploading them to social networks. Phimp.me stands for Photos - Images - Pictures. It is developed at FOSSASIA together with a global community.

    Setting up the Android ProjectÂ¶

    1. Download the phimpme-android project source. You can do this either by forking and cloning the repository (recommended if you plan on pushing changes) or by downloading it as a ZIP file and extracting it.

    2. Install the NDK in Android Studio.

    3. Open Android Studio, you will see a Welcome to Android window. Under Quick Start, select Import Project (Eclipse ADT, Gradle, etc.)

    4. Navigate to the directory where you saved the phimpme-android project, select the root folder of the project (the folder named âphimpme-androidâ), and hit OK. Android Studio should now begin building the project with Gradle.

    5. Once this process is complete and Android Studio opens, check the Console for any build errors.

      • Note: If you receive a Gradle sync error titled, âfailed to find â¦â, you should click on the link below the error message (if available) that says Install missing platform(s) and sync project and allow Android studio to fetch you what is missing.

    6. Download this OpenCV-android-sdk zip file and extract it.

      • Copy all the files from âOpenCV-android-sdk/sdk/native/3rdpartyâ to âphimpme-android/app/src/main/3rdpartyâ (create directory if it doesnât exist)

      • Copy all the files from âOpenCV-android-sdk/sdk/native/staticlibsâ to âphimpme-android/app/src/main/staticlibsâ (create directory if it doesnât exist)

      • Now build your project. If your build fails then try deleting these build directories âphimpme-android/app/.externalNativeBuildâ and âphimpme-android/app/buildâ, if they exist and run the build again.

    7. If build error still persist, try replacing ndk with ndk version 18b from ndk-archives . Once all build errors have been resolved, you should be all set to build the app and test it.

    8. To Build the app, go to Build > Make Project (or alternatively press the Make Project icon in the toolbar).

    9. If the app was built successfully, you can test it by running it on either a real device or an emulated one by going to Run > Run âappâ or pressing the Run icon in the toolbar.
